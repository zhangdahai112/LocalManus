# LocalManus: 全能型 AI Agent 平台 - 产品需求文档 (PRD)

## 1. 项目概述
LocalManus 是一个多模态、工具增强的“通用 Agent”平台，旨在将自然语言指令转化为复杂的数字化产出。与简单的聊天机器人不同，LocalManus 通过编排专业子 Agent，在统一的工作空间内处理文档创建、演示文稿生成、设计任务、调研及数据分析。

### 1.1 目标
为专业人士提供一个无摩擦的“指挥中心”，通过单一 AI 界面执行复杂的创意和分析工作流。

---

## 2. 核心功能模块

### 2.1 "Omnibox" (AI 指挥中心)
平台的核心交互区域。
- **自然语言输入**：支持复杂指令（例如：“分析此 PDF 并将结果转化为 10 页的演示文稿”）。
- **多模态支持**：
    - 文件上传 (+) 以提供上下文（PDF, PPTX, CSV, 图片）。
    - 语音输入（麦克风图标）支持免打字操作。
- **Agent 选择**：手动覆盖以选择特定的 Agent 模式（机器人图标）。

### 2.2 核心能力工具箱 (快捷操作)
针对高频任务的预定义提示词封装：
- **演示文稿**：生成兼容 PowerPoint/Google Slides 的幻灯片。
- **写作**：长文本生成、AI 辅助草拟。
- **设计**：海报、缩略图及包装设计（集成 DALL-E/Midjourney + 类似 Canvas 的布局引擎）。
- **工具类**：
    - **总结**：视频 (YouTube/上传) 及音频转录。
    - **翻译**：文档级翻译 (PDF/Word)。
    - **转换**：视频转网页、PDF 转 Word。

### 2.3 管理与历史 (侧边栏)
- **层级**：项目 -> 文件。
- **最近活动**：快速访问过去的工作，带有状态指示（已完成、进行中）。
- **资源库**：用户上传资产及品牌材料的中央存储。

### 2.4 技能 (Skill) 与沙箱 (Sandbox) 系统 (自动化执行)
- **动态技能调度**：子 Agent 可在运行时从注册表中浏览并“学习”新技能。
- **沙箱执行**：所有自动化操作（文件操作、代码执行）均在隔离环境中进行，以确保系统安全。
- **自动依赖解析**：系统自动安装所需库，以满足 Agent 选择的执行路径。

### 2.5 模板引擎
预优化提示词和布局的展示中心/市场。
- **分类**：市场、调研、设计、教育等。
- **指标驱动**：显示“使用次数”以衡量质量和受欢迎程度。

---

## 3. UI/UX 需求
- **设计风格**：极简“毛玻璃”特效或纯白简约设计。
- **布局**：
    - 固定侧边栏 (240px-280px)。
    - 居中的搜索/操作区（针对“专注”模式）。
    - 用于发现的滚动网格（模板区）。
- **交互元素**：
    - 按钮悬停时的微动画。
    - 针对“重度”AI 任务（新手任务、生成任务）的进度条。

---

## 4. 技术需求 (功能性)
- **状态保存**：用户必须能够返回任何“最近”项目并接续对话，保持上下文完整。
- **实时预览**：生成的幻灯片/文档的实时渲染。
- **沙箱安全**：用户数据与 AI 执行环境严格隔离（多租户安全）。
- **技能扩展性**：开发者可通过标准化 Schema 添加新“技能”，无需重新部署核心 Agent 编排引擎。
- **导出格式**：PDF, PPTX, DOCX, 网页链接, 高清图片。

---

## 5. 用户路径
1. **进入**：用户登录仪表盘。
2. **意图获取**：用户输入“为咖啡店创建商业计划书”或点击“生成幻灯片”。
3. **细化**：AI 提出澄清问题或提供模板选项。
4. **执行**：系统运行专业 Agent（调研 -> 大纲 -> 设计 -> 最终幻灯片）。
5. **迭代**：用户通过聊天或直接操作编辑产出。
6. **交付**：用户导出或分享最终成果。
